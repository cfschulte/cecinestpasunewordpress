And on to using version control - Git

I should have been using version control for a while, but setting it up can be a pain. I don't want to make something visible to the world, especially since some of it might include sensitive data. I've used Subversion in the past but it seems that Git is the big thing these days. So that's what I'm doing.

So far this seems pretty straight forward. I'm not sure where the actual repository is, but this thing is noticing revisions. The question is, what happens when this syncs with my laptop? Will Home Sync barf on the hidden folders? We'll see.

Let's see how this works with symbolic links. Instead of moving index.php and recreating it all the time, I'm going to have everything cecinestpasunewordpress on an archive file and create a symbolic link to the latest one. I guess the thing to do next would be to have a list of the archives along the side.
